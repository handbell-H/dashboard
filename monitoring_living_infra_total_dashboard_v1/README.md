# 생활인프라 종합 모니터링 대시보드 v1

생활인프라 접근성·공급수준·서비스인구 지표를 시군구 단위로 시각화하는 웹 대시보드입니다.
첫 번째 버전을 완성하며, 분석 파이프라인과 시각화를 하나로 묶는 작업이 생각보다 재미있었습니다.

데이터랑 html 다운받아서 실행하시면 실행 됩니다.

---

## 프로젝트 구조

```
monitoring_living_infra_total_dashboard_v1/
├── dashboard.html        # 메인 대시보드 (단일 HTML, 외부 의존성 없음)
├── build.py              # 데이터 → 대시보드 빌드 스크립트
└── data/
    ├── access_index.*        # 접근성 지수 (shapefile)
    ├── supply_index.*        # 공급수준 지수 (shapefile)
    ├── service_pop_index.*   # 서비스인구 지수 (shapefile)
    ├── composite_index.*     # 종합 지수 (shapefile)
    └── *_meta.json           # 각 지수별 메타데이터
```

데이터는 QGIS 플러그인에서 산출 → `build.py`로 대시보드에 내장하는 방식입니다.

## 할 일 (금주 내 수행 목표)

### 1. QGIS 플러그인 수정 (데이터 산출 로직)

박사님과의 논의 결과, 현재 **공급수준 지표가 생활인프라 편리성 지수와 음의 관계**를 보이는 문제를 확인했습니다. 원하는 방향과 반대이므로 아래와 같이 수정 예정입니다.

| 항목 | 현재 | 수정 |
|------|------|------|
| 공급수준 정의 | — | 거주지 면적(1km²) 당 시설 수 |
| 변환 방법 | — | 표준화 전 로그/반로그 변환 적용 (극단값 분포 완화) |
| 표준화 방법 | — | Z-score 산출 후 **T점수 = 50 + 10Z** 변환 (시계열 비교 가능) |

> T점수 표준화는 시점 간 비교를 안정적으로 할 수 있는 장점이 있어 채택했습니다.

### 2. 대시보드 기능 업데이트

- **지도**: 도(광역시·도) 단위 확대 버튼 추가
- **지도**: 지역 클릭 시 해당 지역으로 줌인
- **순위 비교**: 시·도 → 시·군·구 단계적 선택 기능 추가
